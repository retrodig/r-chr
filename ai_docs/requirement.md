# 要件定義

## プロジェクト概要

**r-chr** は NES（ファミコン）の CHR タイルデータを編集するデスクトップアプリケーション。
YY-CHR（Windows 用定番 CHR エディタ）と同等の機能を Rust + egui でクロスプラットフォームに実装することを目指す。

---

## コア機能要件

### ファイル入出力

| 機能 | 状態 |
|------|------|
| 新規作成（空 16KB CHR データ） | 実装済み |
| `.nes` ファイルの読み込み（iNES 形式） | 実装済み |
| `.bin` ファイルの読み込み（生 CHR バイナリ） | 実装済み |
| `.zip` ファイルの読み込み（内部 .nes を展開） | 実装済み |
| `.nes` / `.bin` への上書き保存 | 実装済み |
| `.nes` / `.bin` / `.zip` のドラッグ & ドロップ | 実装済み |
| `.png` / `.bmp` のインポート（CHR に変換） | 実装済み |
| `.png` / `.bmp` のドラッグ & ドロップ | 実装済み |
| `.pal` ファイルの読み込み（マスターパレット差し替え） | 実装済み |
| `.dat` ファイルの読み込み / 保存（DAT パレット） | 実装済み |

### CHR 表示

| 機能 | 状態 |
|------|------|
| CHR データの全体表示（128px 幅、16 タイル/行） | 実装済み |
| バンク切り替え（4KB 単位） | 実装済み |
| パレットセット切り替え（4 セット） | 実装済み |
| 選択ブロックサイズ切り替え（8 / 16 / 32 / 64 / 128 px） | 実装済み |
| アドレスジャンプ（16 進数入力） | 実装済み |

### ドット編集

| 機能 | 状態 |
|------|------|
| ピクセルグリッド編集（8×8 タイル） | 実装済み |
| 描画色の選択（0〜3 の 4 色） | 実装済み |
| スポイトツール | 実装済み |
| アンドゥ（Cmd+Z / Ctrl+Z） | 実装済み |

### パレット編集

| 機能 | 状態 |
|------|------|
| DAT パレット（4 セット × 4 色）の表示・編集 | 実装済み |
| マスターパレット（NES 64 色）からの色選択 | 実装済み |
| デフォルトパレットへのリセット | 実装済み |

### UI / UX

| 機能 | 状態 |
|------|------|
| ダークモード / ライトモード切り替え | 実装済み |
| macOS ネイティブメニューバー | 実装済み |
| Windows / Linux egui メニューバー | 実装済み |
| 日本語フォント（Noto Sans JP） | 実装済み |
| ウィンドウアイコン | 実装済み |
| ウィンドウ枠のダークモード（macOS） | 実装済み |
| 未保存変更の確認ダイアログ | 実装済み |
| About ダイアログ | 実装済み |
